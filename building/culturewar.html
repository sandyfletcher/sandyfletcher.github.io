Inspiration

There's something very special about moving out, living unencumbered by the parental supervision perpetually around your neck from the day you're born. It's common for people to use it as an opportunity to let loose, eating ice cream for breakfast and liquor for lunch. I used this absence of oversight to allocate a larger chunk of my day to computer games.

If someone said that time was wasted, they'd only be mostly right. Imagine a critic admitting they'd only seen a handful of movies prior to getting this movie-reviewing job... would it be reasonable to question the value of their insights? How could they possibly analyze the peaks and valleys of film as a medium when they've only dipped their toe in one part of the river? Surely someone who spent the majority of their day immersed in various games, discussing their intricacies, would've gleaned more, right? Well, fingers crossed.

A couple months back I found myself reminiscing about this period, living in squalor but loving it, and a particular game came to mind â€” Galcon (short for "Galactic Conquest"). It was an online multiplayer space battle, akin to Risk but in real-time, and our frat-without-the-weird-stuff household frequently hosted these celestial throwdowns. At the time I gave it little thought, but in retrospect the website worked really well. There was an abundance of players, matchmaking was speedy, and the game had nailed that addictive balance between skill and luck. Unfortunately for us the game's popularity eventually ended up making it less accessible, and only years later do I understand why.

The adage "there's no such thing as a free lunch" is the easiest way to understand something ostensibly permanent like a website slipping out of existence. Computers use electricity, and typically there's a cost to acquiring that electricity. When you access a website, you're essentially asking for info off someone else's computer, and they're the ones who pick up the tab to process your request. If all you're requesting is a tiny bit of data like today's temperature reading, that's not a very large bill, but Galcon was serving hundreds of concurrent users, and those users were requesting a low-latency game against opponents around the world instead of a two-digit number. The developers chose to pivot from the website we loved to an app with an initial purchase cost, an approach which would easily offset the anticipated computing costs and give them some short-term operational flexibility. It's a logical decision, but it's one that inevitably curtails the number of new players discovering the game moving forward, and the march towards obscurity is much quicker when the landscape is one of downhill trends.

This project intends to reignite interest and evaluate whether Galcon has the strategic depth worthy of joining Chess, Go, and DoTA2 as an AI battleground. I'll begin by attempting to recreate the game as faithfully as possible, ideally so that someone who's never experienced it can get caught up and enjoy the action. From there I'll convert it into an arena for LLMs to test their mettle, pitting various models against one another as another data point in our attempt to benchmark intelligence. That will likely be the extent of the project, but if the results are intriguing I may use this as an avenue to investigate machine learning and attempt to craft the ultimate pilot for galactic conquest.

Initial Steps

At this point in my coding journey I don't have those complex tools at my disposal. The only programming language I'm familiar with capable of making conditional choices is JavaScript, so that'll be what powers both the game architecture and the robot opponents within. Since I plan to have this be a website rather than a locally-running program, I started by setting up a shell that holds the site contents using HTML and CSS. For simplicity's sake, I've arranged it to be inflexible, in the sense that it'll essentially display the same on web as on mobile, and it won't compatible with landscape mode. As you can probably deduce from the site you're reading and the look of the game itself, I'm partial to minimalism. The aesthetics may only be a personal preference, but from a practical perspective fewer distractions means people understand how to proceed more intuitively, so if something's onscreen let's try to ensure it has a clear purpose.

After setting up menus, fiddling with fonts, and ensuring there's visual consistency across platforms and viewport sizes, we're ready to conceptualize how to recreate the game itself. The crux of the gameplay is that planets generate troops at a rate relative to their physical size, and these troops can be dispatched to neighbouring planets to secure new planets that produce even more troops. Naturally, some planets become contested as multiple players vie for its production capabilities, and troops are traded through mutual attrition. Victory is achieved when one player has completely eliminated all opposing players, or given to the player in the dominant position (most planets controlled) when the timer runs out. From a coding perspective, a lot of this is surprisingly simple when broken down into smaller components. We'll need:

Once these guardrails are in place, we can design the order and methodology for planet placement. The starting planets for each player are the logical jumping-off point, so we'll prioritize those and worry about neutral planets afterwards. 


generateHumanPlayerPlanets(humanPlayers) {
    const planets = [];
    const width = this.canvas.width;
    const height = this.canvas.height;
    // Divide the map into quadrants
    const quadrants = [
        { minX: 0, maxX: width/2, minY: height/2, maxY: height },   // Bottom left
        { minX: width/2, maxX: width, minY: 0, maxY: height/2 },    // Top right
        { minX: 0, maxX: width/2, minY: 0, maxY: height/2 },        // Top left
        { minX: width/2, maxX: width, minY: height/2, maxY: height } // Bottom right
    ];
    // For each human player, select a random quadrant and position
    for (let i = 0; i < humanPlayers.length; i++) {
        const player = humanPlayers[i];
        // Select a random quadrant (prefer bottom left for first human player)
        const quadrantIndex = i === 0 ? 0 : Math.floor(Math.random() * quadrants.length);
        const quadrant = quadrants[quadrantIndex];
        // Create buffer space from edges (using player-specific border buffer)
        const buffer = this.config.PLAYER_BORDER_BUFFER + this.config.STARTING_PLANET_SIZE;
        // Find a valid position within the quadrant
        let x, y, valid = false, attempts = 0;
        while (!valid && attempts < this.config.MAX_ATTEMPTS) {
            // Generate position with some randomness but biased toward the center of the quadrant
            const quadrantCenterX = (quadrant.minX + quadrant.maxX) / 2;
            const quadrantCenterY = (quadrant.minY + quadrant.maxY) / 2;
            // Get random position with bias toward quadrant center (60% toward center, 40% random)
            const centerBias = 0.6;
            x = quadrant.minX + buffer + (Math.random() * centerBias + (1 - centerBias) * 0.5) * 
                (quadrant.maxX - quadrant.minX - buffer * 2);
            y = quadrant.minY + buffer + (Math.random() * centerBias + (1 - centerBias) * 0.5) * 
                (quadrant.maxY - quadrant.minY - buffer * 2);
            valid = this.isValidPlayerPosition(x, y, this.config.STARTING_PLANET_SIZE, planets);
            attempts++;
        }
        if (valid) {
            const playerPlanet = new Planet(
                x, y,
                this.config.STARTING_PLANET_SIZE,
                this.config.STARTING_TROOPS,
                player.id,
                this.game
            );
            planets.push(playerPlanet);
            // Remove the used quadrant from options
            quadrants.splice(quadrantIndex, 1);
        } else {
            console.warn('Could not place human player planet in valid position');
            // Fallback to a safe position
            const fallbackX = quadrant.minX + (quadrant.maxX - quadrant.minX) / 2;
            const fallbackY = quadrant.minY + (quadrant.maxY - quadrant.minY) / 2;
            const playerPlanet = new Planet(
                fallbackX, fallbackY,
                this.config.STARTING_PLANET_SIZE,
                this.config.STARTING_TROOPS,
                player.id,
                this.game
            );
            planets.push(playerPlanet);
        }
    }
    return planets;
}

To ensure that competitors have the richest vein of possible options, we'll attempt to assign each a compartmentalized personal chunk of the available space. A player is assigned a quadrant, then a series of checks are run until eligible coordinates are found. Once we've placed that player's planet, we move on to the next player. When everyone's positioned, we add the neutral planets using a similar (more random) process.

  • 3) MOVEMENT: MOBILIZE BATTALIONS AND DIFFERENTIATE ARRIVAL LOGIC

It really just involves determining whether the troops in question belong to the same player, and applying addition or subtraction based on that determination. As for troop speed, the original game didn't bother to account for the complexities of orbit, so neither will we to ensure predictability. 


processTroopArrival(movement) {
    const targetPlanet = movement.to;
    if (targetPlanet.owner === movement.owner) { // reinforcing
        targetPlanet.troops += movement.amount;
    } else { // attacking
        const previousTroops = targetPlanet.troops;
        targetPlanet.troops -= movement.amount;
        const defenderLosses = Math.min(previousTroops, movement.amount); // calculate troops lost (both sides)
        const attackerLosses = targetPlanet.troops < 0 ? 0 : movement.amount;
        this.gameState.incrementTroopsLost(defenderLosses); // track losses for statistics
        this.gameState.incrementTroopsLost(attackerLosses);
        if (targetPlanet.troops < 0) { // capture planet if troops < 0
            targetPlanet.owner = movement.owner;
            targetPlanet.troops = Math.abs(targetPlanet.troops);
            this.gameState.incrementPlanetsConquered(); // update game state for statistics
        }
    }
}
