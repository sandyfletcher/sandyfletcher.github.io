HTML

In the 1980s, researchers at CERN were working with thousands of complex documents that all referenced each other - think an early Wikipedia. One employee, computer scientist Tim Berners-Lee, proposed a system built off hypertext - links that allow you to navigate a network of different pages with ease. He later built off this idea to create Hypertext Markup Language (HTML), the structural backbone of the modern internet.

HTML was designed to be easy for non-technical users to understand, so the layout is essay-like. It starts with a to introduce basic information about the site, <body> containing the bulk of the page content, and <footer> to leave the reader with closing information and signatures. To understand better, let’s make a site of our own:

Like all computer code, HTML for your website is just text. Even pictures are just links written as text. That means you don’t need anything fancy to get started writing your code, just do it in any text editor. If you’re using Windows, you can right-click anywhere on the desktop to bring up an option for a new text file.

Let's copy in some code, and lead off by introducing it as an HTML file. Make sure you swap in your name in the header so it displays properly.

<!DOCTYPE HTML>
<html>
<head>
  <title>________'s Website</title>
  <header>
    <h1>Welcome to Websites!</h1>
    <h2>A place to experiment with HTML features.</h2>
	<p>We'll swap this out with a navigation bar later.</p>
  </header>
</head>
<body>
  <b>Can change text scale size.</b></br>
  <ul>
    <li><h1>OH</h1></li>
	<li><h2>ARE</h2></li>
	<li><h3>YOU</h3></li>
	<li><h4>GOING</h4></li>
    <li><h5>down</h5></li>
  </ul>
</body>
<footer>
  <footer id="contact">
  <h2>Contact Us</h2>
  <p><a href="mailto:info@yoursite.com">Email</a>!</p>
</footer>
</html>

First up is the <header> section, where we set the title of the site and use an <h1> tag to make nice big introduction text. Notice how we include a </head> tag to close out this section before moving on to the next.

After closing the header, we're free to open up the main content of the site with a <body> section. This part is punctuated by "paragraph" tags that separate ideas. Once we're out of ideas, we close out the section with </body>. Last part is a simple <footer> where we can include some contact information in case anyone falls in love with our minimalist web design. Once the entire document is done, we close it out with one final </html> to indicate the end of our file.

Now that we have the code written into the text editor, save the file on your Desktop and right-click to rename the file. Text files and HTML files aren't so different, so you can just switch out the suffix from "txt" to "html". You might get a message that you're making things unstable, but that's the nature of experimentation and everyone will survive. Now when you open the file, it won't be an editable text file, it'll be a website in your browser!

If you want to make any changes, simply switch the filetype back to ".txt" and swap out any info you'd like - it's your site! There are also a couple other features that HTML has that are worth exploring:

<section id="about">
  <h2>Breaking Things Into Sections</h2>
    <p>Sections separate parts of the site.</p>
    <img src="https://via.placeholder.com/150">
</section>
<section id="features">
  <h2>Other Features</h2>
    <h3>Headings</h3>
    <p>Using font size to define sections.</p>
    <p><a href="www.a.com" target="_blank">a.com</a></p>
</section>

Setting up tables doesn't seem like it'd be that interesting, but as we learn more coding languages you realize that they like to use these tables as containers that new toys can be stuck into. For now, they're just there to hold data like an online Excel document.

Admittedly, our website is a little ugly. Before we go any further, let's think about what we'd like to add. A little bit of colour would go a long way, and maybe a nicer font. Maybe things would light up when we hover over them, maybe we could even have a dark mode so the website isn't blinding at night. These improvements will require adding new programming languages to our repertoire, but don't be intimidated - all we need to do is dip our toe in the new features, using only the parts we need to execute on our ideas.
